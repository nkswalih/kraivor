# KRV-011: Sign In with JWT Generation

## Overview
Email/password sign in with JWT pair generation and OTP fallback.

## Authentication Flow

### Step 1: Identify
```
POST /api/auth/signin/identify/
{"email": "user@example.com"}
```
Returns: `next_step` (choose_method | verify_email | signup), user_exists, email_verified, methods

### Step 2: Choose Method
- **Password**: POST /api/auth/signin/password/
- **OTP**: POST /api/auth/signin/otp/send/ → POST /api/auth/signin/otp/verify/

## Token Storage
- **Access Token**: Returned in JSON (store in memory - most secure)
- **Refresh Token**: HttpOnly cookie (30-day expiry)

## Security Features
- Redis-based login lockout (5 failures → 15 min)
- Constant-time password comparison
- OTP with 5-minute expiry, rate limiting
- Login event logging

## Endpoints
| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | /signin/identify/ | Email only |
| POST | /signin/password/ | Password verification |
| POST | /signin/otp/send/ | Send OTP |
| POST | /signin/otp/verify/ | Verify OTP |
| POST | /refresh/ | Refresh tokens |
